# 共享技能使用协议

## 目标

统一 `skills/shared/` 中角色、工作流、模板的使用方式，避免平台适配时各自发散。

## 基本原则

- 角色文件定义“职责边界与强规则”。
- 执行手册定义“执行顺序与常见失误”。
- 工作流定义“阶段流程、输入、输出、准入准出”。
- 模板定义“留痕格式与最小字段”。

## 使用顺序

1. 先读取角色文件
2. 再读取对应执行手册
3. 再读取当前阶段工作流
4. 最后选择对应模板生成或更新交付物

## 强制要求

- 任何关键动作都必须能回链到 Issue、Gate 和文档。
- 若角色文件与工作流冲突，以工作流和阶段门规则为准。
- 若模板字段缺失，不得直接省略，必须显式标注“无”或补齐输入。

## 平台适配要求

- Claude / Codex / OpenCode 只能包装入口，不得篡改共享协议含义。
- 平台层新增能力时，必须保持 `skills/shared/` 语义不变。

## 违规信号

- 只使用角色提示词，不读取工作流
- 只做执行，不更新模板与留痕
- 平台层私自改写阶段门或签字规则
