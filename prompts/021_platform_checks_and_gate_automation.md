# 平台检查与 Gate 自动化

## 目标

把 AgentDevFlow 的流程规则从“纸面约束”推进到“平台可检查约束”，无论接入 GitHub、GitLab 还是其他平台，都应该能实现等价的最小自动化检查。

## 为什么必须做

如果没有平台检查层，流程再完整也会逐渐退化为：

- 文档规则存在，但没人检查
- Comment Gate 存在，但没人拦截
- 双阶段交付存在，但代码仍然直接推进
- 多项目并行时，状态漂移无人发现

## 最小检查集合

### 1. 文档阶段检查

- PRD 是否存在
- Tech 是否存在
- 必要时 QA Case Design 是否存在
- 文档是否已回链主 Issue

### 2. 实现阶段检查

- 实现是否回链主 Issue
- 测试报告是否存在
- 产物关联表是否完整
- 文档阶段确认是否已完成

### 3. Issue Comment Gate 检查

- 当前阶段是否存在必需 Comment
- Comment 是否包含证据链接
- Comment 结论和文档状态是否一致

### 4. 状态一致性检查

- Issue 状态与当前 Gate 是否一致
- Todo 状态与 Issue / Memo 是否一致
- Release 状态与 QA 结论是否一致

## 平台无关要求

- 可以通过 CI、PR Check、Workflow、Bot、Hook 或平台 API 实现
- 只要实现效果等价，就符合 AgentDevFlow 约束
- 平台实现细节不得改写共享流程语义

## 失败处理

- 检查失败时必须阻断当前阶段继续推进
- 必须给出明确失败原因
- 必须说明由谁修复、修复后如何重试
