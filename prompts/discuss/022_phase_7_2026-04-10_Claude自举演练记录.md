## 演练中发现的 CI Bug

### Bug 1: job-level if 条件在 push 事件时求值错误

**现象：** `doc-pr-checks.yml` 在 push 触发时直接失败（0s）

**根因：** `github.event.pull_request.title` 在 push 事件中不存在，表达式求值报错

**修复：** 移除 job-level `if` 条件，改为 step-level guard，用 exit 78 跳过

### Bug 2: Glob pattern `[DOC]` 是字符类而非字面量

**现象：** `code-pr-checks.yml` 的 `[DOC]`* pattern 不匹配 `[DOC #1]`

**根因：** Bash glob 中 `[DOC]` 匹配 D/O/C，而非字面 `[DOC]`

**修复：** 使用 `[[ "$TITLE" =~ ^\[DOC ]]` 正则检测

### Bug 3: CI 跳过逻辑 exit 78 仍显示 failure

**现象：** `code-pr-checks.yml` 在检测到 doc PR 时仍显示 failure

**状态：** 仍在调查，可能与 GitHub Actions 的 step conclusion 机制有关
