# 024 adf-prefix 自举部署方案

## 目标

实现 AgentDevFlow 的自动化自举部署，确保：
1. 本项目 `.claude/skills/` 使用 `adf-` 前缀的 development 版本
2. `~/.claude/` 使用稳定 release 版本
3. 两者命名空间隔离，互不干扰

## 架构

```
skills/shared/  ← 唯一源文件
        ↓ PR merge 触发 bootstrap-sync
.claude/skills/adf-*/SKILL.md  ← 本项目自动更新的 dev 版
        ↓ release 脚本
~/.claude/skills/  ← 全局稳定版
```

## bootstrap-sync 逻辑

### Step 0: 清空目标目录

删除 `.claude/skills/` 下所有已有内容，确保每次同步干净。

### Step 1: 动态发现 Skills

扫描 `skills/shared/`：
- 顶层 `.md` 文件 → skill 名 = 文件名（如 `agent-bootstrap.md`）
- 子目录 → skill 名 = 目录名（如 `agents/`、`workflows/`）

生成映射：
```python
{
    "agent-bootstrap": "adf-agent-bootstrap",
    "create-agent": "adf-create-agent",
    "start-agent-team": "adf-start-agent-team",
    "team-setup": "adf-team-setup",
    "agents": "adf-agents",
    "workflows": "adf-workflows",
    ...
}
```

### Step 2: 生成 SKILL.md

源文件 → 目标路径：
```
skills/shared/agent-bootstrap.md
        ↓
.claude/skills/adf-agent-bootstrap/SKILL.md
```

### Step 3: 内容替换

生成时进行批量引用替换：
1. SKILL.md 内 `name` 字段 → `adf-agent-bootstrap`
2. 全文替换互相引用：
   - `/agent-bootstrap` → `/adf-agent-bootstrap`
   - `/start-agent-team` → `/adf-start-agent-team`
   - `skills/shared/agent-bootstrap` → `.claude/skills/adf-agent-bootstrap`
3. 所有子目录下的 skills 互相引用都要替换

### Step 4: 提交

```
git add .claude/skills/
git commit -m "chore: sync adf-prefix skills"
git push
```

## 命名规范

- 使用 dash（-），不使用 underscore（_）
- 示例：`adf-agent-bootstrap`、`adf-start-agent-team`

## 实现清单

1. 重写 `bootstrap-sync.py`：
   - Step 0: 清空 `.claude/skills/`
   - Step 1: 动态扫描 `skills/shared/` 生成 adf 映射
   - Step 2: 生成 SKILL.md 到 `.claude/skills/adf-*/`
   - Step 3: 批量替换内部引用
   - Step 4: 直接 commit 到 main

2. 更新 `bootstrap-sync.yml`：
   - 目标目录改为 `.claude/skills/`
   - 直接 commit 不创建 PR

3. 确认 `.claude/` 目录在 git 中

4. 测试完整流程

## 禁止

- 不维护两份 skill 代码
- 不在 `skills/shared/` 源文件加 adf 前缀
- 不让本项目调用 `~/.claude/` 下的 release 版本
