# 014 团队启动协议、事件总线与 Agent 执行手册二次对照审计

## 审计背景

在继续对照 `hedge-ai` 的 `prompts/V3.0` 与 `.claude/skills` 后，当前项目仍然存在一个明显问题：虽然中文规则层已经补出大量治理机制，但共享执行层还偏“概念规范”，没有把团队启动、统一创建 Agent、事件总线、运行态状态同步写成足够明确的共享入口。

这会导致：

- 角色定义看起来像职责说明，不像执行手册
- 平台自动化没有统一的共享事件模型
- 启动项目时缺统一的团队启动骨架
- Agent 创建契约停留在 prompt 规则，没有下沉到共享层

## 对照基准

重点参考：

- `hedge-ai/.claude/skills/TEAM_SETUP.md`
- `hedge-ai/.claude/skills/SKILL_PROTOCOL.md`
- `hedge-ai/.claude/skills/EVENT_BUS.md`

- `hedge-ai/.claude/skills/AGENTS/*.md`

## 发现的问题

### 1. 团队启动入口缺少明确的执行顺序

已有 `skills/shared/team-setup.md`，但它更像 kickoff 说明，不像可直接照着执行的统一入口。

### 2. 缺共享事件总线文件

此前有 issue、comment、platform checks、artifact linkage、handoff 等规则，但缺统一的事件模型来把这些串起来。

### 3. Agent 创建统一入口没有下沉到共享层

此前只有 `prompts/zh-cn/006_agent_creation_contract.md` 的规则，没有一个共享层入口告诉平台适配层应如何真正创建角色。

### 4. 个别角色仍偏抽象

虽然角色文件已经有必读、初始化、恢复、禁止行为，但在“激活后先做什么”“如何和 workflow / template 绑定”上仍可更具体。

## 审计结论

本轮补强后，共享层会更接近一个“可执行的通用研发编排底座”而不是“规则文档集合”，但仍有后续缺口：

1. 还没有真实的 CI / Bot / Webhook 落地实现
2. 平台检查目前仍以模板和示例为主
3. 项目实时状态板还没有可运行的数据源
4. 英文镜像尚未同步

## 后续优先级

1. 把事件总线映射到平台适配层的最小自动化样例
2. 增加 issue / comment / gate 同步的伪实现或脚本骨架
3. 补更接近运行态的项目状态板与 Gate 队列样例
