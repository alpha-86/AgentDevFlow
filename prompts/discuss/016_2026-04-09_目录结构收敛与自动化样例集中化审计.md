# 016 目录结构收敛与自动化样例集中化审计

## 审计背景

随着共享角色、workflow、template、适配器样例不断增加，仓库开始出现另一个风险：相似的运行态样例如果继续散落在不同目录，后续会再次变乱。

## 当前问题

1. 平台样例主要散落在各自适配器目录，缺少跨平台样例集中区
2. 用户读 README 时能看到很多样例，但不容易区分哪些是共享骨架、哪些是平台私有说明
3. 仓库结构虽然已经比早期清晰，但“自动化样例放哪里”这个问题还没完全定型

## 本轮调整

### 目录收敛

新增：

- `adapters/examples/`

用于集中放：

- 跨平台的自动化检查骨架
- Gate / Issue Comment / 平台检查的示例 workflow

### 新增样例

- `adapters/examples/gate-check-workflow.example.yml`
- `adapters/examples/issue-comment-check.example.yml`

## 审计结论

这次调整的目标不是新增一个“更花的目录”，而是明确：

- `adapters/{platform}/` 放平台说明和平台样例
- `adapters/examples/` 放跨平台自动化骨架
- `skills/shared/` 继续只放共享语义和模板，不直接承载平台实现

这样目录职责会更稳定，也更方便后续继续补可执行脚本。
