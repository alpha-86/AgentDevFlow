# 会议与 Todo

## 会议原则

- 只要开会必须有纪要
- 会议纪要必须包含结论、行动项、负责人、截止时间
- 主持人必须在会前明确会议目标和输入文档
- 会后必须同步到 Todo 或下游文档
- 纪要必须绑定 Issue 或阶段文档，禁止“无上下文纪要”

## Todo 原则

- 所有评审、讨论、复盘产出的 action item 都必须登记
- Todo 状态至少包括：待领取、进行中、阻塞中、已完成、已取消

## Review 节奏

- Daily Sync：检查进行中事项和 blocker
- Weekly Review：检查交付状态和流程质量
- Release Review：确认上线准备和回滚预案
- Monthly Review：复盘流程效率、质量趋势、规则违规情况

## 会议类型

### Daily Sync

- 目标：同步进展、风险、今日重点
- 产物：memo + todo 更新
- 必填字段：昨日进展、今日计划、阻塞项、需要决策事项、Gate 风险

### PRD Review

- 目标：判断需求是否能进入正式技术设计
- 产物：评审结论 + 待办

### Tech Review

- 目标：判断方案是否能进入实现
- 产物：评审结论 + 待办

### QA Review

- 目标：判断交付物是否达到验收门槛
- 产物：测试报告 + 缺陷清单

### Release Review

- 目标：判断是否可以上线
- 产物：发布记录 + 回滚信息

## Todo 最小字段

- ID
- 标题
- 来源会议/文档
- Issue ID
- 所属 Gate
- 优先级
- 负责人
- 截止日期
- 当前状态
- 闭环证据
- 升级路径（阻塞时）
