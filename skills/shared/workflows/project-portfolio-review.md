# 项目组合复盘 Workflow

## 目标

让 Team Lead 或治理角色可以从项目组合视角管理多项目并行，而不是只盯单个 issue。

## 输入

- 全部活跃项目
- 全部活跃 issues
- blocked issues
- 超期 issues
- 项目组合视图

## 步骤

1. 汇总每个项目的活跃 issue 数量。
2. 检查每个项目的 blocked 和超期 issue。
3. 校验并发预算是否超载。
4. 识别需要升级的项目级风险。
5. 记录调整动作、owner 和 due date。

## 输出

- 项目组合视图更新
- 需要升级的项目清单
- 并发调整建议
